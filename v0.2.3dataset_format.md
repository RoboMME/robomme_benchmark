# Robomme 数据集格式说明

本文档描述 `RecordWrapper.py` 录制生成的 HDF5 数据集格式（当前版本：`robomme-0.2b3-rpy4`）。

---

## 1. 文件与目录结构

```
<dataset_root>/
  hdf5_files/
    <EnvID>_ep<Episode>_seed<Seed>.h5   # 每个 episode 一个文件
  videos/                               # （可选）录制视频
```

- 录制由 `RobommeRecordWrapper` 完成。
- **只有成功的 episode 会写入 HDF5**，失败的 episode 数据会在 `close()` 时丢弃。

---

## 2. HDF5 层级总览

```
├─ env_<EnvID>/
│  ├─ episode_<N>/
│  │  ├─ record_timestep_0/          # 每个时间步一个 group
│  │  │  ├─ obs/                     # 观测数据
│  │  │  ├─ action/                  # 动作数据
│  │  │  └─ info/                    # 元信息
│  │  ├─ record_timestep_1/
│  │  │  └─ ...
│  │  ├─ ...
│  │  └─ setup/                      # Episode 级别配置
│  └─ episode_<M>/
│     └─ ...
```

---

## 3. 各字段详解

### 3.1 `obs/` —— 观测数据

| 字段 | 类型 / 形状 | 说明 |
|---|---|---|
| `front_camera_rgb` | `uint8 (H, W, 3)` | 正面相机 RGB 图像 |
| `wrist_camera_rgb` | `uint8 (H, W, 3)` | 腕部相机 RGB 图像 |
| `front_camera_depth` | `float (H, W, 1)` | 正面相机深度图 |
| `wrist_camera_depth` | `float (H, W, 1)` | 腕部相机深度图 |
| `joint_state` | `float64 (9,)` | 机器人关节角度（7 自由度 + 2 夹爪），不足 9 维则零填充 |
| `eef_state` | `float64 (6,)` | 末端执行器状态 `[x, y, z, roll, pitch, yaw]`（连续化 RPY） |
| `gripper_state` | `float64 (2,)` | 夹爪开合度（`qpos` 最后两维） |
| `gripper_close` | `bool` | 夹爪是否闭合（任一维 < 0.03 即为 `True`） |
| `eef_velocity` | `float64 (6,)` | 末端执行器线速度 (3) + 角速度 (3) |
| `front_camera_segmentation` | `int (H, W, ...)` | 正面相机原始分割掩码 |
| `front_camera_segmentation_result` | 同上 | 经子目标过滤后的分割结果 |
| `front_camera_extrinsic` | `float (4, 4)` | 正面相机外参矩阵（OpenCV 约定） |
| `wrist_camera_extrinsic` | `float (4, 4)` | 腕部相机外参矩阵（OpenCV 约定） |

> **RPY 连续化**：`eef_state` 中的 RPY 由 `rpy_util.build_endeffector_pose_dict` 生成，经过四元数符号对齐 + RPY unwrap 处理，保证帧间连续。

### 3.2 `action/` —— 动作数据

| 字段 | 类型 / 形状 | 说明 |
|---|---|---|
| `joint_action` | `float64 (8,)` 或 `str "None"` | 关节空间动作（7 自由度 + 1 夹爪指令）；不足 8 维则以 `-1` 填充；无动作时存字符串 `"None"` |
| `eef_action` | `float64 (7,)` | 末端执行器动作 `[x, y, z, roll, pitch, yaw, gripper]`，其中 gripper 取自 `joint_action` 的最后一维 |
| `keypoint_action` | `float64 (7,)` | 关键点动作 `[x, y, z, roll, pitch, yaw, gripper]`。每帧都写入，**持久化上一次的 keypoint 值**；首个 keypoint 出现前为全零 |

#### `keypoint_action` 说明

- **keypoint 持久化**：keypoint 刷新后，后续帧仍然复用最新值，直到下一个 keypoint 出现。
- `is_keyframe`（见 info）标记了 keypoint 被刷新的帧。

### 3.3 `info/` —— 元信息

| 字段 | 类型 | 说明 |
|---|---|---|
| `record_timestep` | `int` | 当前帧在 episode 内的编号 |
| `simple_subgoal` | `bytes (UTF-8)` | 当前子目标名称（Planner 视角） |
| `simple_subgoal_online` | `bytes (UTF-8)` | 当前子目标名称（Online 视角） |
| `grounded_subgoal` | `bytes (UTF-8)` | 带分割对象占位符填充后的子目标文本 |
| `grounded_subgoal_online` | `bytes (UTF-8)` | 同上（Online 视角） |
| `is_video_demo` | `bool` | 该帧是否属于视频演示（Demonstration）阶段 |
| `is_keyframe` | `bool` | 该帧是否为 keypoint 刷新帧（`True` 表示该帧出现了新 keypoint） |

---

## 4. `setup/` —— Episode 级别配置

每个 `episode_<N>/setup/` 下保存一次性配置信息：

| 字段 | 类型 | 说明 |
|---|---|---|
| `seed` | `int` | 环境随机种子 |
| `difficulty` | `str` | 难度等级 |
| `task_goal` | `str` | 自然语言任务描述 |
| `front_camera_intrinsic` | `float (3, 3)` | 正面相机内参矩阵（取首帧） |
| `wrist_camera_intrinsic` | `float (3, 3)` | 腕部相机内参矩阵（取首帧） |
| `subgoal_list/` | *group* | 子目标列表 |
| `subgoal_list/subgoal_<i>_name` | `bytes` | 第 i 个子目标名称 |
| `subgoal_list/subgoal_<i>_demonstration` | `bool` | 第 i 个子目标是否为演示阶段 |

---

## 5. 重要实现细节

1. **RPY 连续化**（`rpy_util.py`）：四元数 → 归一化 → 符号对齐 → RPY 主值 → unwrap，保证帧间平滑无跳变。Keypoint 的 RPY 也走同一流水线，以当前帧的 eef 缓存做对齐，但**不写回缓存**（缓存仅由 eef 路径更新）。
2. **Failsafe**：当 episode 步数 ≥ 2000 时强制截断，标记为失败。
3. **相机内参**：每 episode 仅保存一次（取首帧），存放于 `setup/` 下。外参则每帧保存（存放于 `obs/`）。
4. **joint_state 维度保证**：始终为 9 维（7 关节 + 2 夹爪），不足时零填充。
5. **joint_action 维度保证**：始终为 8 维（7 关节 + 1 夹爪指令），不足时 `-1` 填充。
